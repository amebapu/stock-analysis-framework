# 米勒维尼投资分析框架 v5.0 (100分制)

> **版本**: v5.0  
> **创建时间**: 2026-03-09  
> **核心方法论**: 马克·米勒维尼《股市魔法师》SEPA策略  
> **设计目的**: 对博主推荐标的进行独立第三方验证，辅助建仓决策  
> **仓位上限**: 单标的最高20%  
> **数据源**: stock-data v2.3.0（唯一数据源，腾讯自选股）  
> **计算层**: Python calc_indicators.py（确定性计算，零幻觉）

---

## 一、核心设计原则

### 1.1 层层证伪，而非寻找买入理由
- 先检查是否触碰红线（排除）
- 再评估各维度得分
- 最后给出仓位建议

### 1.2 数据准确性原则
- **禁止估算** - 不能"大概"、"应该"、"可能"
- **禁止编造** - 没有的数据不能说"估计是XXX"
- **明确标注** - 无法获取的数据写"未获取"
- **诚实告知** - 如果不能100%确定，明确告诉用户

### 1.3 数据架构（两层设计）

| 层级 | 组件 | 用途 | 说明 |
|------|------|------|------|
| 数据层 | stock-data v2.3.0 | K线、行情、财务、新闻、评级、筹码 | 唯一数据源（腾讯自选股） |
| 计算层 | Python calc_indicators.py | MA/RSI/MACD/SEPA/量价 | 确定性计算，零第三方依赖 |

**设计理由**：
- 数据真 + 算法对 = 结果对
- 禁止大模型"心算"技术指标（存在幻觉、精度丢失、不可复现风险）
- Python 脚本使用标准库，同一输入永远得到同一输出

## ⚠️ 重要：股票代码格式规范（2026-03-09 教训）

### 美股代码格式要求

**关键教训**：不同接口对美股代码格式要求不同，**格式错误会导致数据获取失败**。

| 数据类型 | 正确格式 | 错误格式 | 示例 |
|----------|----------|----------|------|
| **K线数据** | `usTSLA`, `usAAPL` | `TSLA`, `AAPL` | `stock-data kline usTSLA day 200 qfq` |
| **财务数据** | `TSLA.N`, `TSLA.O`, `TSLA.OQ` | `usTSLA`, `TSLA` | `stock-data finance TSLA.N income 4` |
| **实时行情** | `usTSLA` | `TSLA.N` | `stock-data quote usTSLA` |
| **筹码分布** | `usTSLA` | `TSLA.N` | `stock-data chip usTSLA` |
| **新闻研报** | `usTSLA` | `TSLA.N` | `stock-data news usTSLA 1 10 2` |

### 常见错误

❌ **错误**：`stock-data finance usSNDK income 4`  
✅ **正确**：`stock-data finance SNDK.N income 4` 或 `stock-data finance SNDK.O income 4`

### 快速检查方法

如果财务数据返回空或报错，立即尝试以下格式：
```bash
# 格式1：.N 后缀（纽约证交所）
stock-data finance SNDK.N income 4

# 格式2：.O 后缀（纳斯达克）
stock-data finance SNDK.O income 4

# 格式3：.OQ 后缀（纳斯达克全拼）
stock-data finance SNDK.OQ income 4
```

## 二、评分体系（总分100分）

### 2.1 第一层：大盘环境（仅建议，不参与评分）

**目的**：判断市场环境，给出整体仓位建议

| 检查项 | 标准 | 数据获取方式 |
|--------|------|-------------|
| 标普500趋势 | >MA50且>MA200 | `stock-data kline usSPY day 200 qfq \| python calc_indicators.py` |
| VIX恐慌指数 | <20为舒适区 | `stock-data quote usVIX` (读取 price 字段) |

**输出示例**：
- "大盘环境良好，建议整体仓位80%"
- "大盘环境一般，建议整体仓位50%"
- "大盘环境差，建议空仓观望"

**注意**：大盘环境仅作为参考建议，不影响个股评分

---

### 2.2 第二层：基本面（25分）

**数据来源**：`stock-data finance <code> income/cashflow`

#### 2.2.1 基础评分项

| 指标 | 量化标准 | 分值 | 数据来源 |
|------|----------|------|----------|
| 季度净利润增长 | ≥25% | 10分 | income |
| 营收增长 | ≥20% | 5分 | income |
| ROE | ≥17% | 5分 | income |
| 经营现金流 | 为正 | 5分 | cashflow |

#### 2.2.2 扣分项（从25分中扣除）

| 扣分项 | 触发条件 | 扣分 |
|--------|----------|------|
| 净利润同比下滑 | 最近季度<去年同期 | -5分 |
| 营收同比下滑 | 最近季度<去年同期 | -5分 |
| 连续2季度亏损 | 净利润为负 | -10分 |

**计算示例**：
```
基础分: 25分
- 净利润增长15% (<25%): -5分
- 营收增长-3% (<20%): -5分  
- ROE 4.9% (<17%): -5分
- 现金流为正: +0分
= 25 - 5 - 5 - 5 = 10分
但给保底分15分（不扣除超过10分）
最终: 15/25分
```

#### 2.2.3 硬门槛
- 基本面<15分 → **D级，直接排除**
- 理由：米勒维尼强调盈利增长是股价上涨的发动机

### 2.3 第三层：技术面（60分）

**数据来源**：K线数据 → Python `calc_indicators.py` + `stock-data chip`

#### 2.3.1 SEPA趋势模板（40分）

**获取方式**：`stock-data kline <code> day 200 qfq | python scripts/calc_indicators.py`

| 检查项 | 标准 | 分值 | 计算方式 |
|--------|------|------|----------|
| 股价>MA50 | 站上短期均线 | 8分 | Python calc_indicators.py |
| MA50>MA150>MA200 | 多头排列 | 8分 | Python calc_indicators.py |
| MA200走平或上升 | 对比20日前MA200 | 5分 | Python calc_indicators.py |
| 股价>MA200 | 中长期趋势确认 | 8分 | Python calc_indicators.py |
| 距52周高点<25% | 非深度回调 | 6分 | Python calc_indicators.py |
| 距52周低点>25% | 脱离底部确认 | 5分 | Python calc_indicators.py |

> **降权说明**：SEPA从v4.1的50分降为40分，因部分检查项存在共线性（如"股价>MA200"与"MA50>MA150>MA200"高度相关），腾出空间给新增的RSI/MACD/量价/筹码维度。

**硬门槛**：
- SEPA模板<4项通过 → **最高C级（60分），建议观望**
- 米勒维尼：仅第二阶段（主升浪）具备交易价值

#### 2.3.2 RSI 相对强弱（5分）

**获取方式**：同上（`calc_indicators.py` 一次输出全部指标）

| RSI(14)范围 | 得分 | 说明 |
|-------------|------|------|
| 40-70 | 5分 | 健康区间（既未超买也未超卖） |
| 30-40 或 70-80 | 3分 | 边缘区间 |
| <30 或 >80 | 0分 | 超卖/超买，风险偏高 |
| N/A | 不参与评分 | 数据不足（需≥15根K线） |

#### 2.3.3 MACD 趋势动量（5分）

**获取方式**：同上

| MACD信号 | 得分 | 说明 |
|----------|------|------|
| 多头（DIF>DEA，柱状图为正） | 5分 | 强势上涨动能 |
| 多头减弱/空头减弱 | 3分 | 过渡阶段 |
| 空头（DIF<DEA，柱状图为负） | 0分 | 下行动能 |
| N/A | 不参与评分 | 数据不足（需≥35根K线） |

#### 2.3.4 量价配合（5分）

**获取方式**：同上（`calc_indicators.py` 的量价分析模块）

| 指标 | 得分标准 | 分值 |
|------|----------|------|
| 涨日均量/跌日均量 | 比值>1.3（上涨放量下跌缩量） | 3分 |
| 量比（最新量/20日均量） | 0.8~1.5 为正常 | 2分 |

#### 2.3.5 筹码结构（5分）

**获取方式**：`stock-data chip <code>`

| 指标 | 得分标准 | 分值 |
|------|----------|------|
| 获利比例 | >60%（多数持仓者获利） | 2分 |
| 70%筹码集中度 | <15%（筹码集中） | 2分 |
| 股价>平均成本 | 当前价>avgCost | 1分 |

> **市场降权**：A股满分5分；港股/美股筹码维度降权为满分3分（机构比例高，筹码理论有效性降低）。

#### 2.3.6 美股数据降级规则

美股盘中/收盘后 K 线 API 可能数据受限，按以下规则降级：

| K线数量 | 数据完整性 | 处理方式 | 技术面上限 |
|---------|-----------|---------|-----------|
| ≥200根 | A级 | 正常计算所有指标 | 60分 |
| 50~199根 | B级 | MA200标N/A，其余正常 | 50分 |
| 3~49根 | C级 | MA/RSI/MACD全部N/A | 20分（仅chip+量价） |
| ≤2根 | C级 | 技术面全部N/A | 0分 |

> 报告中必须标注数据完整性等级（A/B/C），让用户清楚技术面评分的可信度。

### 2.4 第四层：催化剂（15分）

**数据来源**：`stock-data news/rating/report/quote/asfund/hkfund`

| 指标 | 量化标准 | 分值 | 数据获取 |
|------|----------|------|----------|
| 新闻情绪 | 近10条新闻中正面≥3条（含"增长/突破/超预期/新高"等关键词） | 3分 | `stock-data news <code> 1 10 2` |
| 无重大利空 | 近10条无"下调/警告/违规/退市/爆雷/做空"等负面关键词 | 3分 | `stock-data news <code> 1 10 2` |

| 资金流向 | A股: 主力净流入>0且近3日流入; 港股: 卖空占比下降 | 3分 | `stock-data asfund`(A股) / `stock-data hkfund`(港股) |

## 三、风控红线（一票否决）

触碰任一红线，直接排除或止损：

| 红线 | 触发条件 | 处理 |
|------|----------|------|
| 跌破MA200 | 技术破位，趋势反转 | 排除 |
| 买入后跌7-8% | 触及止损位 | 无条件止损 |
| 连续2季度业绩下滑 | 基本面恶化 | 排除 |
| 重大负面新闻 | 黑天鹅事件 | 排除 |

**米勒维尼的铁律**：
- 单笔最大亏损 ≤ 总资金1%
- 止损幅度 7-8%（硬性）
- 盈亏比 ≥ 1:3

## 四、评分等级与仓位分配

### 4.1 总分计算

```
总分 = 基本面(25分) + 技术面(60分) + 催化剂(15分) = 100分
```

### 4.2 评级与仓位

**仓位上限**：单标的最高20%

| 总分 | 等级 | 建议 | 仓位 | 说明 |
|------|------|------|------|------|
| 90-100 | A+ | **强烈建议** | **20%** | SEPA完美+基本面优秀+有催化剂 |
| 80-89 | A | **可以建仓** | **15%** | 满足硬门槛，技术面良好 |
| 70-79 | B | **小仓位试探** | **10%** | 有瑕疵，需严格止损 |
| 60-69 | C | **建议观望** | **5%** | 某项硬门槛未达标或SEPA<4项 |
| 0-59 | D | **明确回避** | **0%** | 触碰红线或多项不合格 |

### 4.3 仓位使用规则

- **A+ (20%)**：可满仓该标的，对框架有极高信心
- **A (15%)**：标准仓位，正常建仓
- **B (10%)**：试探性仓位，严格止损，随时准备离场
- **C (5%)**：极轻仓或放弃，风险收益比不佳
- **D (0%)**：不参与，保存现金

### 4.4 硬门槛速查

| 条件 | 结果 |
|------|------|
| 基本面<15分 | D级，排除 |
| SEPA模板<4项通过 | 最高C级，观望 |
| 触碰红线 | D级，排除/止损 |

## 五、Python 技术指标计算脚本

### 5.1 calc_indicators.py

**路径**：`scripts/calc_indicators.py`（项目根目录下的 scripts 文件夹）

**功能**：从 stdin 读取 stock-data kline 的 JSON 输出，精确计算以下指标：
- **MA**: MA50 / MA150 / MA200（简单移动平均线）
- **RSI(14)**: Wilder 平滑法
- **MACD(12,26,9)**: DIF / DEA / 柱状图 + 多空信号
- **SEPA 趋势模板**: 6 项检查（通过/失败/N/A）
- **量价分析**: 20日均量、放量缩量、涨跌日量比

**特性**：
- 零第三方依赖（仅 json/sys/math 标准库）
- 自动兼容 v2.2.2+ nodes 格式和旧版数组格式
- 数据不足时输出 N/A（标注需要多少根 K 线），绝不编造
- 内置数据完整性评级（A/B/C 级）

**使用方法**：
```bash
# A股 - 200日前复权K线
stock-data kline sh600519 day 200 qfq | python scripts/calc_indicators.py

# 美股 - 200日前复权K线
stock-data kline usAAPL day 200 qfq | python scripts/calc_indicators.py

# 港股
stock-data kline hk00700 day 200 qfq | python scripts/calc_indicators.py

# 大盘 SPY 检查
stock-data kline usSPY day 200 qfq | python scripts/calc_indicators.py
```

### 5.2 技术指标计算公式

| 指标 | 公式 | 最少所需K线 |
|------|------|-----------|
| MA(N) | SMA = Σ(最近N个收盘价) / N | N 根 |
| RSI(14) | RS = avg_gain / avg_loss, RSI = 100 - 100/(1+RS) (Wilder平滑) | 15 根 |
| MACD | DIF = EMA(12) - EMA(26), DEA = EMA(DIF, 9), 柱 = 2*(DIF-DEA) | 35 根 |
| EMA(N) | EMA_t = price × k + EMA_(t-1) × (1-k), k = 2/(N+1) | N 根 |

## 六、完整分析流程

### 步骤1：大盘环境检查（建议）
```bash
# SPY 趋势（MA50/MA200 判断）
stock-data kline usSPY day 200 qfq | python scripts/calc_indicators.py

# VIX 恐慌指数
stock-data quote usVIX
```
→ 输出环境建议，不参与个股评分

### 步骤2：个股数据获取
```bash
# 1. 实时价格
stock-data quote <code>

# 2. K线 + 全部技术指标（MA/RSI/MACD/SEPA/量价 一次算完）
stock-data kline <code> day 200 qfq | python scripts/calc_indicators.py

# 3. 筹码分布
stock-data chip <code>

# 4. 财务数据（最近4期）
stock-data finance <code> income 4        # 美股
stock-data finance <code> lrb             # A股
stock-data finance <code> zhsy 4          # 港股

# 5. 现金流
stock-data finance <code> cashflow 4      # 美股
stock-data finance <code> xjll            # A股/港股

# 6. 新闻资讯
stock-data news <code> 1 10 2

# 7. 机构评级/研报
stock-data rating <code>                  # A股
stock-data report <code>                  # 港美股

# 8. 资金流向
stock-data asfund <code>                  # A股
stock-data hkfund <code> day 20           # 港股
```

### 步骤3：逐层评分

| 层级 | 满分 | 评分依据 |
|------|------|----------|
| 基本面 | 25分 | stock-data finance |
| 技术面 | 60分 | Python calc_indicators.py（SEPA 40 + RSI 5 + MACD 5 + 量价 5 + 筹码 5） |
| 催化剂 | 15分 | stock-data news/rating/report/quote/asfund/hkfund |

### 步骤4：计算总分，确定等级和仓位

```
总分 = 基本面 + 技术面 + 催化剂
↓
检查硬门槛（基本面<15 → D级; SEPA<4项 → 最高C级）
↓
确定等级（A+/A/B/C/D）
↓
确定仓位（20%/15%/10%/5%/0%）
```

## 八、使用示例（TSLA分析）

### 8.1 数据获取
```bash
# 获取所有数据
stock-data quote usTSLA
stock-data kline usTSLA day 200 qfq | python scripts/calc_indicators.py
stock-data chip usTSLA
stock-data finance usTSLA income 4
stock-data finance usTSLA cashflow 4
stock-data news usTSLA 1 10 2
stock-data report usTSLA
```

### 8.2 数据分析

**基本面（15/25分）**：
- 净利润增长: -47% (❌, -5分) [来源: finance]
- 营收增长: -3% (❌, -5分) [来源: finance]
- ROE: 4.9% (❌, -5分) [来源: finance]
- 现金流: 为正 (✅, +5分) [来源: finance]
- 小计: 25-5-5-5+5 = 15分（保底）

**技术面（38/60分）**：
- SEPA模板: 4/6项通过 → 28/40分 [来源: Python]
  - 股价>MA50: $396<$428 ❌
  - MA50>MA150>MA200: $428>$414>$392 ✅
  - MA200走平或上升: ✅
  - 股价>MA200: $396.73>$392.10 ✅
  - 距高点<25%: 19% ✅
  - 距低点>25%: 85% ✅
- RSI(14): 41 (健康区间) → 5/5分 [来源: Python]
- MACD: 空头减弱 → 3/5分 [来源: Python]
- 量价配合: 量能平稳 → 2/5分 [来源: Python]
- 筹码: 港美股降权 → 0/3分（美股筹码数据有限） [来源: chip]
- 小计: 28+5+3+2+0 = 38分

**催化剂（8/15分）**：
- 新闻情绪: 正面2条/10（不足3条）→ 0/3分 [来源: news]
- 无重大利空: ✅ → 3/3分 [来源: news]
- 机构评级: 研报偏正面 → 2/3分 [来源: report]
- 流动性: 日成交远超$50M → 3/3分 [来源: quote]
- 资金流向: N/A（美股无此数据）→ 0/3分
- 小计: 0+3+2+3+0 = 8分

**总分**: 15+38+8 = 61分 (C级，建议观望)

### 8.3 结论
- 评级: C级
- 仓位: 5%（极轻仓或放弃）
- 原因: 基本面恶化（净利润-47%，营收-3%），触碰"连续业绩下滑"红线边缘；技术面尚可但股价跌破MA50
- 建议: 建议观望，等待下季度财报验证是否回暖
