# 米勒维尼投资分析框架 v5.2 (100分制)

> **版本**: v5.2  
> **创建时间**: 2026-03-09  
> **最后更新**: 2026-03-10T1200  
> **核心方法论**: 马克·米勒维尼《股市魔法师》SEPA策略  
> **设计目的**: 对博主推荐标的进行独立第三方验证，辅助建仓决策  
> **仓位上限**: 单标的最高20%  
> **数据源**: stock-data v2.3.0（唯一数据源，腾讯自选股）  
> **计算层**: Python calc_indicators.py v1.3.0（确定性计算，零幻觉）  
> **强制要求**: 252根K线优先 / 缺项映射 / 大盘环境必查 / 有效满分+置信等级输出

## 一、核心设计原则

### 1.1 层层证伪，而非寻找买入理由
- 先检查是否触碰红线（排除）
- 再评估各维度得分
- 最后给出仓位建议

### 1.2 数据准确性原则
- **禁止估算** - 不能"大概"、"应该"、"可能"
- **禁止编造** - 没有的数据不能说"估计是XXX"
- **明确标注** - 无法获取的数据写"未获取"
- **诚实告知** - 如果不能100%确定，明确告诉用户

### 1.3 数据架构（两层设计）

| 层级 | 组件 | 用途 | 说明 |
|------|------|------|------|
| 数据层 | stock-data v2.3.0 | K线、行情、财务、新闻、评级、筹码 | 唯一数据源（腾讯自选股） |
| 计算层 | Python calc_indicators.py | MA/RSI/MACD/SEPA/量价 | 确定性计算，零第三方依赖 |

**设计理由**：
- 数据真 + 算法对 = 结果对
- 禁止大模型"心算"技术指标（存在幻觉、精度丢失、不可复现风险）
- Python 脚本使用标准库，同一输入永远得到同一输出

## ⚠️ 重要：股票代码格式规范（2026-03-09 教训）

### 美股代码格式要求

**关键教训**：不同接口对美股代码格式要求不同，**格式错误会导致数据获取失败**。

| 数据类型 | 正确格式 | 错误格式 | 示例 |
|----------|----------|----------|------|
| **K线数据** | `usTSLA`, `usAAPL` | `TSLA`, `AAPL` | `stock-data kline usTSLA day 200 qfq` |
| **财务数据** | `TSLA.N`, `TSLA.O`, `TSLA.OQ` | `usTSLA`, `TSLA` | `stock-data finance TSLA.N income 4` |
| **实时行情** | `usTSLA` | `TSLA.N` | `stock-data quote usTSLA` |
| **筹码分布** | `usTSLA` | `TSLA.N` | `stock-data chip usTSLA` |
| **新闻研报** | `usTSLA` | `TSLA.N` | `stock-data news usTSLA 1 10 2` |

### 常见错误

❌ **错误**：`stock-data finance usSNDK income 4`  
✅ **正确**：`stock-data finance SNDK.N income 4` 或 `stock-data finance SNDK.O income 4`

### 快速检查方法

如果财务数据返回空或报错，立即尝试以下格式：
```bash
# 格式1：.N 后缀（纽约证交所）
stock-data finance SNDK.N income 4

# 格式2：.O 后缀（纳斯达克）
stock-data finance SNDK.O income 4

# 格式3：.OQ 后缀（纳斯达克全拼）
stock-data finance SNDK.OQ income 4
```

## 二、评分体系（总分100分）

### 2.1 第一层：大盘环境（仅建议，不参与评分）

**输出示例**：

- "大盘环境差，建议空仓观望"

**注意**：大盘环境仅作为参考建议，不影响个股评分

### 2.2 第二层：基本面（25分）

**数据来源**：`stock-data finance <code> income/cashflow`

#### 2.2.1 基础评分项

| 指标 | 量化标准 | 分值 | 数据来源 |
|------|----------|------|----------|
| 季度净利润增长 | ≥25% | 10分 | income |
| 营收增长 | ≥20% | 5分 | income |
| ROE | ≥17% | 5分 | 见下方分市场说明 |
| 经营现金流 | 为正 | 5分 | 见下方分市场说明 |

#### 2.2.2 ROE/经营现金流 分市场获取链路 (v5.2)

| 市场 | ROE | 经营现金流 |
|------|-----|-----------|
| **A股** | `stock-data finance <code> summary` → 直接取 `roe_weighted` | `stock-data finance <code> xjll` → 经营现金流净额 |
| **港股** | `stock-data finance <code> zhsy` + `zcfz` → `ROE = 净利润 / 股东权益 × 100%`（近似值） | `stock-data finance <code> xjll <N>` → 经营活动产生现金净流量 |
| **美股** | `stock-data finance <code>.N/.O income <N>` + `balance <N>` → `ROE = 净利润 / 股东权益 × 100%`（近似值）<br>⚠️ 美股不支持 `summary`，禁止假设存在统一 ROE 字段 | `stock-data finance <code>.N/.O cashflow <N>`<br>⚠️ **已知限制**: 当前 stock-data cashflow 路径存在间歇性异常（`type param error`），框架必须兜底 |

**获取失败处理（统一规则）**：
- ROE 获取失败 → 标记"未获取"→ 从基本面满分中剔除5分（满分变为20分）
- 经营现金流获取失败 → 标记"未获取"→ 从基本面满分中剔除5分
- 两者均失败 → 基本面满分变为15分
- **分值映射**: `映射后分数 = 实际得分 / 实际满分 × 100`，确保缺项不导致评级不公平

#### 2.2.3 扣分项（从25分中扣除）

| 扣分项 | 触发条件 | 扣分 |
|--------|----------|------|
| 净利润同比下滑 | 最近季度<去年同期 | -5分 |
| 营收同比下滑 | 最近季度<去年同期 | -5分 |
| 连续2季度亏损 | 净利润为负 | -10分 |

**计算示例**：
**计算公式**（必须展示计算过程）：
```
基础分: 25分
- 净利润增长 XX% (<25%): [-5/+10]分
- 营收增长 XX% (<20%): [-5/+5]分
- ROE XX% (<17%): [-5/+5]分（若无法获取，此项剔除）
- 现金流: [+0/+5]分（若无法获取，此项剔除）
= 25 - 扣分 + 加分 = XX分
（保底分15分，不扣除超过10分）
最终: XX/25分（或XX/20分，若某项剔除）
```

**计算示例**：
```
基础分: 25分
- 净利润增长15% (<25%): -5分
- 营收增长-3% (<20%): -5分  
- ROE 4.9% (<17%): -5分
- 现金流为正: +0分
= 25 - 5 - 5 - 5 + 0 = 10分
但给保底分15分（不扣除超过10分）
最终: 15/25分
```

#### 2.2.3 硬门槛
- 基本面<15分 → **D级，直接排除**
- 理由：米勒维尼强调盈利增长是股价上涨的发动机

| 指标 | 得分标准 | 分值 |
|------|----------|------|
| 涨日均量/跌日均量 | 比值>1.3（上涨放量下跌缩量） | 3分 |
| 量比（最新量/20日均量） | 0.8~1.5 为正常 | 2分 |

#### 2.3.5 筹码结构（5分A股/3分港美股, v5.2确定性打分）

**获取方式**：`stock-data chip <code>`

**确定性评分规则**（使用 `score_chip()` 函数）：

| 子项 | 权重 | 满分条件 | 半分条件 | 零分条件 |
|------|------|----------|----------|----------|
| 获利比例(profitPercent) | 40% | ≥60% | 40-60% | <40% |
| 70%筹码集中度(p70) | 40% | <15% | 15-25% | ≥25% |
| 股价>平均成本(avgCost) | 20% | 股价>avgCost | — | 股价≤avgCost |

**满分基准**：A股5分、港股/美股3分（机构比例高，筹码理论有效性降低）。

**缺失处理**：子项数据缺失时，该子项权重从分母中剔除，按剩余有效权重计算。全部缺失则筹码维度不参与评分。

#### 2.3.6 美股数据降级规则

美股盘中/收盘后 K 线 API 可能数据受限，按以下规则降级：

| K线数量 | 数据完整性 | 处理方式 | 技术面上限 |
|---------|-----------|---------|-----------|
| ≥252根 | A级 | 正常计算所有指标 | 60分 |
| 220~251根 | A-级 | 正常计算，但52周高低点可能不完整 | 58分 |
| 200~219根 | B级 | MA200趋势判断标N/A | 55分 |
| 50~199根 | C级 | MA200标N/A，RSI/MACD正常 | 45分 |
| 35~49根 | D级 | RSI正常，MACD/MA200标N/A | 30分 |
| 15~34根 | E级 | 仅RSI可用 | 15分 |
| 3~14根 | F级 | MA/RSI/MACD全部N/A | 5分（仅chip） |
| ≤2根 | F级 | 技术面全部N/A | 0分 |

> 报告中必须标注数据完整性等级（A/B/C），让用户清楚技术面评分的可信度。

## 三、风控红线（一票否决）

触碰任一红线，直接排除或止损：

| 红线 | 触发条件 | 处理 |
|------|----------|------|
| 跌破MA200 | 技术破位，趋势反转 | 排除 |
| 买入后跌7-8% | 触及止损位 | 无条件止损 |
| 连续2季度业绩下滑 | 基本面恶化 | 排除 |
| 重大负面新闻 | 黑天鹅事件 | 排除 |

**米勒维尼的铁律**：
- 单笔最大亏损 ≤ 总资金1%
- 止损幅度 7-8%（硬性）
- 盈亏比 ≥ 1:3

## 四、评分等级与仓位分配

### 4.1 总分计算（含分值映射, v5.2）

**步骤1：计算各维度实际得分，记录缺失项**
```
基本面实际得分 = XX分 / 实际满分(25/20/15分)  [缺失项: ...]
技术面实际得分 = XX分 / 实际满分(≤60分)        [缺失项: ...]
催化剂实际得分 = XX分 / 15分
原始总分 = 基本面 + 技术面 + 催化剂
有效满分 = 基本面满分 + 技术面满分 + 15
```

**步骤2：分值映射到100分制**
```
映射后总分 = (原始总分 / 有效满分) × 100

示例1（现金流未获取）：
  基本面：18/20分，技术面：50/55分(筹码N/A)，催化剂：12/15分
  原始总分 = 80, 有效满分 = 90
  映射后 = 80/90 × 100 = 88.9分 (A级)
  缺失项: [经营现金流, 筹码结构]
```

**步骤3：确定置信等级**
```
置信等级 = f(K线完整性等级, 分母覆盖率)
├─ 高:   A/A- 且覆盖率 ≥ 90%
├─ 中:   B且覆盖率≥80%, 或 A/A-且覆盖率70-89%
├─ 低:   C/D 或覆盖率 60-79%
└─ 极低: E/F 或覆盖率 < 60%
```

**步骤4：基于映射后分数评级**
- 评级标准始终以100分制为基准
- 映射确保数据缺失不会导致评级不公平
- 缺项处理只针对"无法获取/无法计算"；"已获取但不达标"保留失分

### 4.2 评级与仓位

**仓位上限**：单标的最高20%

| 总分 | 等级 | 建议 | 仓位 | 说明 |
|------|------|------|------|------|
| 90-100 | A+ | **强烈建议** | **20%** | SEPA完美+基本面优秀+有催化剂 |
| 80-89 | A | **可以建仓** | **15%** | 满足硬门槛，技术面良好 |
| 70-79 | B | **小仓位试探** | **10%** | 有瑕疵，需严格止损 |
| 60-69 | C | **建议观望** | **5%** | 某项硬门槛未达标或SEPA<4项 |
| 0-59 | D | **明确回避** | **0%** | 触碰红线或多项不合格 |

### 4.3 仓位使用规则

- **A+ (20%)**：可满仓该标的，对框架有极高信心
- **A (15%)**：标准仓位，正常建仓
- **B (10%)**：试探性仓位，严格止损，随时准备离场
- **C (5%)**：极轻仓或放弃，风险收益比不佳
- **D (0%)**：不参与，保存现金

### 4.4 硬门槛速查

| 条件 | 结果 |
|------|------|
| 基本面<15分 | D级，排除 |
| SEPA模板<4项通过 | 最高C级，观望 |
| 触碰红线 | D级，排除/止损 |

## 五、Python 技术指标计算脚本

### 5.1 calc_indicators.py

**路径**：`scripts/calc_indicators.py`（项目根目录下的 scripts 文件夹）

**功能**：从 stdin 读取 stock-data kline 的 JSON 输出，精确计算以下指标：
- **MA**: MA50 / MA150 / MA200（简单移动平均线）
- **RSI(14)**: Wilder 平滑法
- **MACD(12,26,9)**: DIF / DEA / 柱状图 + 多空信号
- **SEPA 趋势模板**: 5 项检查（通过/失败/N/A），每项含得分和有效满分
- **量价分析**: 20日均量、放量缩量、涨跌日量比
- **筹码打分**: `score_chip()` 确定性函数，支持市场降权
- **分值映射**: `calculate_score_mapping()` 缺项剔除后映射到100分
- **置信等级**: `determine_confidence()` K线完整性 + 覆盖率联合判定

**特性**：
- 零第三方依赖（仅 json/sys/math 标准库）
- 自动兼容 v2.2.2+ nodes 格式和旧版数组格式
- 数据不足时输出 N/A（标注需要多少根 K 线），绝不编造
- 内置数据完整性评级（A/A-/B/C/D/E/F 级）
- v5.2: 输出有效满分、缺失项列表、置信等级和映射得分

**使用方法**：

#### Linux/macOS
```bash
# A股 - 252日前复权K线（满足所有指标计算需求）
stock-data kline sh600519 day 252 qfq 2>/dev/null | sed '/^\[HTTP/d' | python calc_indicators.py

# 美股 - 252日前复权K线
stock-data kline usAAPL day 252 qfq 2>/dev/null | sed '/^\[HTTP/d' | python calc_indicators.py

# 港股
stock-data kline hk00700 day 252 qfq 2>/dev/null | sed '/^\[HTTP/d' | python calc_indicators.py

# 大盘 SPY 检查（美股大盘）
stock-data kline usSPY day 252 qfq 2>/dev/null | sed '/^\[HTTP/d' | python calc_indicators.py
```

#### Windows PowerShell（v1.3.0 起无需 sed）
```powershell
# 脚本自动过滤 [HTTP] 日志行，无需 sed 和 2>/dev/null
stock-data kline sh600519 day 252 qfq | python calc_indicators.py

# 美股含筹码评分
stock-data chip usSNDK > sndk_chip.json
stock-data kline usSNDK day 252 qfq | python calc_indicators.py --chip sndk_chip.json --market US

# 港股含筹码评分

```

#### 命令行参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--chip` | 筹码JSON文件路径（stock-data chip 输出） | 无（不含筹码） |
| `--market` | 市场类型: A/HK/US | A |

### 5.2 技术指标计算公式与K线需求

| 指标 | 公式 | 最少所需K线 | 说明 |
|------|------|-----------|------|
| MA50 | SMA = Σ(最近50个收盘价) / 50 | **50根** | 基础均线 |
| MA150 | SMA = Σ(最近150个收盘价) / 150 | **150根** | 中期趋势 |
| MA200 | SMA = Σ(最近200个收盘价) / 200 | **200根** | 长期趋势 |
| RSI(14) | Wilder平滑法 | **15根** | 相对强弱 |
| MACD(12,26,9) | EMA(12)-EMA(26), DEA=EMA(DIF,9) | **35根** | 趋势动量 |
| 52周高低点 | max(highs)/min(lows) | **252根** | 一年区间(v1.3.0改用high/low) |

**关键结论**：
- **最佳调用**: `stock-data kline <code> day 252 qfq`（满足所有指标）
- **最低可用**: 50根（可算 MA50、RSI、MACD、量价；SEPA 部分项可算）
- **不足时处理**: 不可计算的指标标 N/A，同时从有效满分中剔除对应分值
- **v5.2变更**: 移除"MA200趋势判断"（原需220根），不再要求220根才能完成SEPA

## 八、使用示例（TSLA分析）

### 8.1 数据获取
```bash
# 1. 大盘环境（必须获取）
stock-data kline usSPY day 252 qfq 2>/dev/null | sed '/^\[HTTP/d' | python calc_indicators.py

stock-data news usTSLA 1 10 2
stock-data report usTSLA
```

### 8.2 数据分析

**基本面（15/25分）计算过程**：
```
基础分: 25分
- 净利润增长: -47% (<25%): -5分
- 营收增长: -3% (<20%): -5分
- ROE: 4.9% (<17%): -5分
- 现金流: 为正: +5分
= 25 - 5 - 5 - 5 + 5 = 15分
（保底15分，已满足）
最终: 15/25分
```
详细项：
- 净利润增长: -47% (❌, -5分) [来源: finance]
- 营收增长: -3% (❌, -5分) [来源: finance]
- ROE: 4.9% (❌, -5分) [来源: finance]
- 现金流: 为正 (✅, +5分) [来源: finance]

**技术面（38/60分）计算过程**：
```
SEPA模板: 3/5项通过 → 24/40分
RSI: 41 (健康区间40-70) → 5/5分
MACD: 空头减弱 → 3/5分
量价: 量能平稳 → 2/5分
筹码: 港美股降权 → 0/3分(美股满分3分)
= 24 + 5 + 3 + 2 + 0 = 34/58分(筹码有效满分3分→技术面满分58)
```
详细项：
- SEPA模板: 3/5项通过 → 24/40分 [来源: Python]
  - 股价>MA50: $396<$428 ❌ [0/8分]
  - MA50>MA150>MA200(多头排列): $428>$414>$392 ✅ [8/8分]
  - 股价>MA200: $396.73>$392.10 ✅ [8/8分]
  - 距高点<25%: 19% ✅ [8/8分]
  - 距低点>25%: 85% ✅ [8/8分] (注: 本例股价虽>MA200但<MA50，实际通过4项非3项; 此处仅为格式示例)
- RSI(14): 41 (健康区间) → 5/5分 [来源: Python]
- MACD: 空头减弱 → 3/5分 [来源: Python]
- 量价配合: 量能平稳 → 2/5分 [来源: Python]
- 筹码: 港美股降权 → 0/3分（美股筹码数据有限） [来源: chip]

**催化剂（8/15分）计算过程**：
```
新闻情绪: 2条/10 (<3条): 0/3分
无重大利空: ✅: 3/3分
机构评级: 正面为主: 2/3分
流动性: >$50M: 3/3分
资金流向: N/A: 0/3分（美股无此数据）
= 0 + 3 + 2 + 3 + 0 = 8/15分
```
详细项：
- 新闻情绪: 正面2条/10（不足3条）→ 0/3分 [来源: news]
- 无重大利空: ✅ → 3/3分 [来源: news]
- 机构评级: 研报偏正面 → 2/3分 [来源: report]
- 流动性: 日成交远超$50M → 3/3分 [来源: quote]
- 资金流向: N/A（美股无此数据）→ 0/3分

**总分计算（含映射, v5.2）**：
```
有效满分 = 25分(基本面全有) + 58分(技术面,筹码3分) + 15分 = 98分
原始总分 = 15 + 34 + 8 = 57分
映射后分数 = 57/98 × 100 = 58.2分
分母覆盖率 = 98/100 = 98%
K线完整性 = [取决于实际K线数量]
置信等级 = [取决于K线等级+覆盖率]
评级: D级 (58.2分, <60分区间)
```

若现金流未获取示例：
```
有效满分 = 20分(剔除现金流) + 58 + 15 = 93分
原始总分 = 15 + 34 + 8 = 57分
映射后分数 = 57/93 × 100 = 61.3分
评级: C级 (分数更公平)
缺失项: [经营现金流]
```

### 8.3 结论
- 评级: C级
- 仓位: 5%（极轻仓或放弃）
- 原因: 基本面恶化（净利润-47%，营收-3%），触碰"连续业绩下滑"红线边缘；技术面尚可但股价跌破MA50
